# 阶段 B：优化

主流程步骤 6–9。仅在用户确认继续优化后进入；改代码前须确认项目范围。

## 项目范围（步骤 6 前须明确）

| 用户意图 | 判定 | 行为 |
|----------|------|------|
| 当前工作区 | 「当前项目」「本项目」；或本地 URL 端口与 `package.json` 一致 | 在工作区读代码、出清单、改代码 |
| 其他项目 | 步骤 6 Other 填路径；或消息里已带路径 | 在该路径操作；路径无效 → 通用建议 |
| 只看报告 | 选「到此为止」；或一开始说「只分析」 | 不进入阶段 B |

- **禁止**：未进入阶段 B 就搜代码、改文件。
- **禁止**：因 URL 域名与工作区不同就假定「仅审计」而跳过步骤 5（除非用户已明确不要优化）。
- 其他项目路径：步骤 6 Other 一次收集；路径不存在则搜索常见目录，复述后请用户确认。
- 「涉及文件」写已确认项目根的相对路径，或标注「需在目标项目中自行定位」。
- 步骤 6 细则见 [askquestion.md](askquestion.md#步骤-6--确认目标项目范围)。

## 优化清单如何得出

**阶段 A** 可附简要问题摘要：有证据的问题 + 改法**方向**，**无「涉及文件」**。

**阶段 B** 完整清单：

1. **来源 = 阶段 A 证据**：fail/warn audit、Trace Insight，无需重跑（除非回归）。
2. **LCP 四段对症下药**（LCP = TTFB + 资源加载延迟 + 加载时长 + 渲染延迟）：
   - **TTFB 占比大** → 后端/缓存/CDN，不是图片。
   - **资源加载延迟大** → `preload` + `fetchpriority="high"`，首屏勿 `lazy`。
   - **资源加载时长大** → AVIF/WebP、`srcset`、压缩。
   - **渲染延迟大** → 关键 CSS 内联、异步 CSS、拆分/延迟 JS。
3. **定位代码**（已确认项目根）：结合 audit 资源/DOM/请求定位文件；选「通用建议」则不写文件路径。
4. **修复方案**：「问题 → 改法 → 涉及文件」。
5. **优先级**：
   - **P0**：LCP > 4s、CLS > 0.25、A11y fail、SEO 关键缺失、首屏 `lazy`、TBT 明显超标。
   - **P1**：`preload`+`fetchpriority`、WebP/AVIF、`font-display:swap`、cache-control、关键 CSS、防 CLS 尺寸。
   - **P2**：代码分割、DB 索引、依赖瘦身；`scheduler.yield()`、Speculation Rules。
6. **忽略不可控项**：第三方 CDN、云抖动 → 归入说明，不进清单。

## 阶段 B 输出格式

```
## 优化清单

**目标项目：** 当前工作区 / <绝对路径> / 通用建议（不定位文件）

### P0 — 必须修复
1. 问题 → 改法 → 涉及文件

### P1 — 强烈建议
2. …

### P2 — 可选优化
3. …
```

每项稳定编号（1、2、3…），供步骤 8 AskQuestion 引用。清单输出后**立即** AskQuestion 步骤 8，不在末尾再列选项。

## 阶段 B 输出边界

| 只输出 | 不要输出 |
|--------|----------|
| P0/P1/P2 清单 + 执行项选择 | 重讲审计教程、lab vs field 科普 |
| 前后指标差分（回归时） | 重讲 skill 流程 |

- 无证据的猜测不写进清单。
- 全绿时写「达标，无必改项」；边际项标「可选」且 ≤ 3 条。
- 未确认项目前不读仓库、不 diff、不 commit。

## 任务边界

- 不自动 `git commit` / `push` / 部署，除非用户明确要求。
- 不修改 `.env`、nginx、云控制台，除非用户授权。
- 不混测 desktop + mobile（除非用户要求对比）。
- 用户选「到此为止」→ 立即收束，不追加 follow-up。

## 改代码与回归

执行完选中项后，按 [regression.md](regression.md) 分流回归。
