# 用户决策交互（AskQuestion）

步骤 5 / 6 / 8 **一律用 AskQuestion**（或等价结构化 UI），**禁止**在正文用「1. … 2. …」或「需要我继续吗？」代替。

- 自由文本（路径、编号、补充说明）→ 用 AskQuestion **Other**，同一次弹窗完成。
- 同一轮回复：先输出正文（报告/清单），**紧接着** AskQuestion；不等下一条消息。
- 无 AskQuestion 时才退化为请用户回复选项 id。

## 步骤 5 — 报告后是否继续（A → B）

**触发**（须同时满足）：

1. 用户未明确「只分析 / 不要改代码 / 只问达标」。
2. 审计报告已输出。
3. 存在 fail 或「需改进」（全绿且未要求优化 → 不弹窗，写「达标，无必改项」）。

**不触发**：用户已明确要优化且已指明项目 → 跳过，进入步骤 6（已指明则复述确认）→ 7。

| 字段 | 值 |
|------|-----|
| `title` | 下一步 |
| `questions[0].id` | `next-step` |
| `questions[0].prompt` | 审计已完成，是否继续生成代码级优化清单？ |
| 选项 1 | `continue` — **继续优化** |
| 选项 2 | `stop` — **到此为止** |

- `continue` → 6（若需）→ 7 → 8
- `stop` → 一句收束
- 用户沉默 → **不要**自动进入阶段 B

## 步骤 6 — 确认目标项目范围

**触发**：选了继续优化，且仓库范围未明。

| 字段 | 值 |
|------|-----|
| `title` | 确认项目范围 |
| `questions[0].id` | `project-scope` |
| `questions[0].prompt` | 优化代码在哪个项目？选「其他项目」时，请在 **Other** 中填写仓库根路径（如 `~/projects/foo`）。 |
| 选项 1 | `workspace` — **当前工作区** |
| 选项 2 | `generic` — **只要通用建议，不定位具体文件** |
| **Other** | 其他项目仓库根路径 → `other` + `path` |

**解析**：

- `workspace` → 当前工作区根
- `generic` → 清单不写「涉及文件」
- Other 填路径 → 规范化、验证存在 → 在该目录读代码；无效则请重选
- Other 空且意图为其他项目 → 不要假设工作区

已指明「本项目」或消息里带路径 → 复述确认，不必弹窗。

## 步骤 8 — 选择要执行的优化项

**触发**（须同时满足）：

1. 已选继续优化（或一开始就明确要优化）。
2. 优化清单已输出（含编号）。
3. 清单有证据的必改或建议项。

**优化项 ≤ 5**：`allow_multiple: true`

| 选项顺序 | 内容 |
|----------|------|
| 1 | `all` — **全部** |
| 2…n | `item-1`、`item-2`…「编号 + 简短标题」 |
| 倒数第 2 | `no` — **不执行，结束** |
| **Other** | 补充说明或额外需求 |

**优化项 > 5**：

| 选项顺序 | 内容 |
|----------|------|
| 1 | `all` — **全部** |
| 2 | `no` — **不执行，结束** |
| **Other** | 填写编号（如 `1,3,5`）或补充需求 |

编号一律在 **Other** 输入，禁止固定选项里放「在对话中回复编号」。

**处理**：

- `all` → 执行全部（须完成步骤 6）
- `item-*` / Other 编号 → 只执行选中项
- Other 自由文本 → 按补充描述执行
- `no` → 停止
- 执行完 → [regression.md](regression.md)
