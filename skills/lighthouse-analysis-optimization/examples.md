# 示例

**例 1 — 只问达标**：「某线上 URL 移动端达标吗？」→ 阶段 A 审计 + 结论；全绿则「达标，无必改项」；**不弹步骤 5**。

**例 2 — 分析外部 URL（默认流）**：「分析用户提供的线上 URL」→ 阶段 A 报告 → **AskQuestion 步骤 5** → 继续 → **步骤 6**（若需）→ 阶段 B 清单 → **步骤 8** → 改代码。

**例 3 — 明确不要优化**：「分析某 URL，不要改代码」→ 阶段 A 报告 → **跳过步骤 5**。

**例 4 — 一开始就要优化本项目**：「分析并优化本项目首页」→ 阶段 A → **跳过步骤 5** → 复述确认工作区 → 阶段 B 清单 → 步骤 8 → 改代码 → 本地回归 / 线上提醒部署。

**例 5 — 其他项目路径已知**：「优化 `~/projects/foo` 下的线上 URL」→ 阶段 A → 确认该路径 → 阶段 B → 在该目录改文件。

**例 6 — 线上 URL 不可达**：完整线上 URL → 重试最多 5 次 → 失败报告并结束；**不** build 工作区、**不**测本地。
